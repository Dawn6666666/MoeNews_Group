适用范围：2023本科人才培养方案

**二、课程的性质与任务**

**课程性质：**本课程是软件工程专业专升本方向的个性化教育限选课，属集中实践课程，培养学生运用企业级开发技术分析、建模、解决复杂计算机工程问题的能力。

**课程任务：**本课程主要通过讲授Java企业级开发的基础知识和面向复杂问题的解决方案等的基础上，重点开展程序设计实际操作的实践教学环节，使学生理论和实践相结合。熟练掌握Java企业级应用程序的开发过程和面向不同工程实际进行的程序设计，在实际操作过程中提升概念、原理、设计和运行等专业操作与设计能力等。通过本课程的学习，学生能够针对具体的复杂计算机工程问题，设计企业级应用并编写高效程序，使学生的实践动手能力得到真正的提升，更好的适应社会对人才的需要，为后续的就业打下坚实基础。

**四、课程目标**

**德育目标：**树立独立思考、分工协作、团结奋进的意识；具有不怕困难，迎难而上，勇于开拓的创新精神；发扬求真务实、辛勤劳动、甘于奉献的科学家精神。

**课程目标1：**根据给定的复杂工程问题，能够审清问题的已知数据和约束条件，明确问题的目标，选择合适的开发技术，设计并实现合理的企业级系统功能；（3-3）

**课程目标2：**能够评价计算机工程问题的实现方案和社会影响因素，并进行总结汇报；通过分组讨论和师生评价分析方案的优缺，制定解决方案的修改计划。（6-2）

**七、课程目标达成评价方法**

课程目标达成评价采用修课学生调查问卷法和课程成绩分析法，修课学生调查问卷法是一种定性评价方式，课程成绩分析法是一种定量评价方法。德育目标的达成评价采用间接法，以调查问卷（学生自评）的形式进行。以课程成绩分析法为主，定性和定量两种评价方法相互印证课程目标的达成情况，详见《计算机与软件学院课程质量评价机制》。

**（一）课程成绩分析法**

课程目标j的达成情况评价值的计算公式如下：

$$D_{j\_ c} = \sum_{i \in E_{j}}^{}{\frac{a_{i}}{t_{i}}{\times p}_{i}}$$

其中，$E_{j}$是第$j$个课程目标的考核方式集合，$a_{i}$是考核方式$i$的平均得分，$t_{i}$是考核方式$i$的目标总分，$p_{i}$是考核方式$i$支撑课程目标$j$的权重。

**（二）修课学生调查问卷法**

针对课程的每一个课程目标$j$，分别统计课程目标达成情况（很好、好、一般、差、很差）的调查问卷份数，每种达成情况的量化值分别为5、4、3、2、1，利用下面公式计算课程目标$j$的达成情况的调查问卷评价值：

$$D_{j\_ w} = \frac{\sum_{i = 1}^{5}{x_{i} \times i}}{\sum_{i = 1}^{5}x_{i}}/5$$

其中，$i$表示课程目标$j$达成情况的量化值,$x_{i}$表示课程目标$j$达成情况等级的调查问卷份数。
