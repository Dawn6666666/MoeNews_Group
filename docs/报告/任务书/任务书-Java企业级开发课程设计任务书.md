## 二、课程设计题目

### 1. 课程选题方式

采用老师给出参考题目或自主选题的形式（贴近生活、真实环境）。

（1）学生可以选择其中的一个参考题目进行设计与开发。

（2）自主选题时，选择与生活相关或有兴趣的题目，先由指导教师进行题目可行性分析，若题目符合课程设计要求，可根据难易程度指定小组人数。

### 2. 开展方式

小组协作方式，一个小组完成一个题目的完整分析、设计与开发过程，由学生自主结合，每组人数≦3 人。

### 3. 参考题目

（1）基于七牛云的云盘系统开发：基于七牛云提供免费对象存储服务，制作一个个人云盘，包括文件上传、下载、改名、删除、共享等。

（2）学生宿舍管理管理系统：查看房屋信息、添加房屋信息、修改和删除房屋信息、查看房屋类型信息、添加房屋类型信息、修改和删除房屋类型信息和修改用户的部分用户信息和注销用户等。

（3）健身房管理系统：健身房有各种不同种类的课程若干，不同的课程计费方式不同，如健身房通常都是收费使用，允许多个用户同时进入课程，而个别课程通常都是独占使用。健身房提供体育器械租赁，体育器械有免费提供的，但入馆要登记，出馆要归还，但不收费；也有计时收费的体育器械，同样要登记和归还。健身房的客户有散客和会员两种。健身房有会员管理，会员可以给会员账号中充值，消费时可以直接从会员账户中扣除，散客进入课程则需要交纳一定的现金作为押金。客户入馆时，创建账单记录记录下课程和时间以及租赁的器械，租赁的器械通过账单附加记录。客户出馆时，根据账单记录和账单附加，自动计算费用。会员能够登陆系统，能够预约课程、查看预约记录、修改密码和修改基本信息。

（4）新闻管理系统：用户管理模块、新闻管理模块、新闻类别管理模块等。

（5）基于 websocket + Spring Boot + RabbitMQ 的在线聊天系统：基于 websocket 实现基于网页的聊天系统。

（6）基于 Spring Boot 的数据库客户端设计与开发：自己完成一个基于 Web 的数据库客户端。

## 三、目的与要求

### 1. 目的

（1）学生能够针对具体问题，设计企业级 Web 应用并编写高效程序，使学生的实践动手能力得到真正的提升，更好的适应社会对人才的需要，为后续的就业打下坚实基础。并撰写文档，培养学生解决复杂工程问题能力的理念和要求，实现本课程支撑课程目标的达成。**（支撑的课程目标 1、2）**

### 2. 要求

（1）能够对有实际背景的 Java 企业级应用项目进行系统需求分析和功能设计、撰写规范的需求分析报告）；

（2）熟练掌握 Web 企业级应用程序的开发过程、MVC 设计模式和面向不同工程实际进行的程序设计，在实际操作过程中提升概念、原理、设计和运行等专业操作与设计能力；

（3）能够熟练选择合适的技术方案进行系统实现和编程。

（4）能够与同学有效沟通，团结协作，有责任有担当，掌握课程设计报告的撰写和修改。

## 四、进度安排

课程设计总时间为 1 周，课程设计的题目和要求应至少提前两周布置，使学生能够对任务有深入的认识，学生选择合适的开发环境，学会使用开发工具。

（1）根据选题内容查阅技术文献资料，分析问题，明确任务（1 天），能够利用网络，对课题相关的参考文献和技术资料进行检索；根据实训选题的要求，充分分析和理解题目，明确问题的要求，确定最终的任务。。

（2）技能实践及项目具体设计（1 天），掌握项目开发所需的专业知识；根据项目需求进行分析和设计；综合考虑各种因素，分析影响效率的因素，提出改进设想。

（3）程序编码、调试与测试（1 天），能够熟练操作程序开发工具；按照设计方案，正确使用相应的专业知识进行工程项目开发；进行系统测试。调试正确后认真整理源程序及注释，形成格式和风格良好的源程序和测试清单。。

（4）撰写课程设计报告（0.5 天），按照规定的格式、采用合适工具，撰写课程实训报告。

（5）课程设计答辩验收（0.5 天），能够顺利运行程序，演示设计结果，并正确、清晰回答教师提出的问题。

## 五、撰写课程设计报告或课程设计总结

总结报告包括需求分析、功能设计、系统实现、总结、参考资料等。报告等要书写整洁、条理清晰、内容全面。
