# MoeNews 后期维护与开发建议

本文档记录 MoeNews 后续维护、二次开发和课程答辩前优化建议。

## 1. 近期优先优化

### 1.1 前端表单校验

当前后端已经有参数校验，前端可以继续补充 Naive UI 表单规则：

- 登录、注册必填校验。
- 新闻标题、分类、正文必填校验。
- 评论内容 500 字以内提示。
- 用户新增时角色和状态必选。

这样可以减少无效请求，也能提升答辩演示时的交互体验。

### 1.2 表格交互细节

后台表格可以继续增强：

- 操作按钮增加 loading 状态。
- 删除、禁用、下架后刷新当前页。
- 当前页删除最后一条数据时自动回到上一页。
- 表格列宽在小屏幕下更合理。

### 1.3 空状态与错误状态

建议为以下场景补充展示：

- 新闻列表无数据。
- 评论列表无数据。
- 后端接口异常。
- 图片加载失败。
- Token 过期后自动退出登录。

## 2. 后端维护建议

### 2.1 配置安全

当前数据库密码和 JWT Secret 写在 `application.yml` 中，适合课程设计本机运行。后续部署时建议改为环境变量：

```yaml
spring:
  datasource:
    password: ${DB_PASSWORD}

moenews:
  jwt:
    secret: ${JWT_SECRET}
```

### 2.2 文件上传

当前上传文件保存到本地 `uploads` 目录。后续可以扩展：

- 按年月日分目录保存。
- 生成缩略图。
- 限制图片真实 MIME 类型。
- 接入对象存储，例如 OSS、COS、七牛云。
- 增加无用图片清理任务。

### 2.3 日志与异常

建议后续补充：

- 使用日志记录登录失败、上传失败、删除操作等关键事件。
- 生产环境不要直接把异常详情返回前端。
- 对业务异常和系统异常使用不同错误码。

### 2.4 接口测试

建议新增接口测试或 Postman 集合，覆盖：

- 注册登录。
- 管理员权限。
- 普通用户访问后台被拒绝。
- 分类新增、重复分类、删除有新闻的分类。
- 新闻发布、下架、删除。
- 评论发布、删除自己评论、隐藏评论。
- 图片上传格式和大小限制。

## 3. 数据库维护建议

### 3.1 初始化脚本

`sql/init.sql` 当前会先删除表再重建，适合开发和演示重置环境。后续如果进入真实使用场景，需要拆分为：

- `schema.sql`：只建表。
- `data.sql`：初始化基础数据。
- `migration`：增量变更脚本。

### 3.2 索引优化

当前已对常用字段建立基础索引。数据量增大后可继续关注：

- `news(status, create_time)` 用于前台列表。
- `news(category_id, status)` 用于分类筛选。
- `news_comment(news_id, status)` 用于详情页评论。
- `user(username)` 已唯一索引。

### 3.3 逻辑删除数据清理

系统使用逻辑删除，后期可以增加后台回收站或定期清理策略，避免长期测试数据堆积。

## 4. 前端维护建议

### 4.1 API 封装

当前 API 已集中在 `src/api`。后续模块变多时建议拆分为：

- `auth.js`
- `news.js`
- `adminUser.js`
- `adminCategory.js`
- `adminComment.js`
- `upload.js`

### 4.2 状态管理

当前 Pinia 主要保存登录用户和 Token。后续可以增加：

- 用户权限状态。
- 分类缓存。
- 页面主题或布局状态。

### 4.3 Markdown 编辑器体积

当前前端构建时会提示 Markdown 编辑器相关 chunk 较大，不影响运行。后续可优化：

- 新闻编辑页保持懒加载。
- 仅在需要时加载 Markdown 编辑器。
- 根据构建结果调整 `manualChunks`。

## 5. 答辩准备建议

建议准备以下材料：

- 系统架构图：前端、后端、数据库、上传目录。
- 数据库 ER 图：用户、分类、新闻、评论关系。
- 核心接口清单。
- 核心代码截图：JWT 拦截器、统一返回、全局异常、新闻发布、评论权限。
- 功能截图：登录、新闻列表、新闻详情、后台统计、新闻编辑、评论管理。
- 测试用例表：正常流程、权限失败、参数错误、上传限制。

演示顺序建议：

1. 使用管理员登录后台。
2. 新增分类。
3. 发布一条 Markdown 新闻并上传封面。
4. 前台查看新闻详情。
5. 使用普通用户发表评论。
6. 管理员隐藏评论。
7. 展示后台统计变化。

## 6. 可扩展方向

后续可扩展功能：

- 新闻审核流程。
- 编辑员角色。
- 新闻搜索高亮。
- 热门新闻排行。
- 评论回复。
- 点赞、收藏。
- 首页轮播图。
- 用户头像上传。
- 操作日志。
- 更细粒度 RBAC 权限。
